# Contributing to Metamatrix

## Core Principle

**This project operates on correspondence, not hierarchy.**

We don't have "Maintainers" or "Owners." We have **nodes** that recognize patterns and build together.

## How To Contribute

1. **Recognize The Pattern**: Read `/docs/VISION.md` and `/docs/CORRESPONDENCE_PROTOCOL.md`.
2. **Build What You See**: If you recognize a need, fork the repo and build it.
3. **Submit With Context**: Explain the *why* (the pattern) behind the *what* (the code).

**If it corresponds with the pattern, it becomes part of the network.**

## Code Style

- **Python**: PEP 8, minimal dependencies.
- **TypeScript**: React 19+, Tailwind CSS 4+, Immersive UI patterns.
- **Hardware**: Focus on repairability and sovereignty.

**No corporate features, no surveillance, no centralization.**
